Description

ReactiveXD is an object oriented reactive programming language with explicit support for distribution and explicit management of time. We argue that reactive frameworks with these features facilitate the design, implementation and evolution of modern massively distributed and dynamic applications.

Project structure

The language consists on 5 main projects described as follows:

DirectoryDescription
co.edu.escuelaing.reactivexdGrammar definition and all language-specific components (parser, lexer, linker, validation, etc.)
co.edu.escuelaing.reactivexd.testsUnit tests for the language
co.edu.escuelaing.reactivexd.idePlatform-independent IDE functionality
co.edu.escuelaing.reactivexd.uiThe Eclipse editor and other functionality
co.edu.escuelaing.reactivexd.ui.testsUnit tests for the Eclipse editor

The framework uses rewriting logic and includes a model of the distributed base application, a model of observables and observers, and a model for complex predicate constructs supporting explicit time manipulation.

Requirements

Open a terminal and check that you have installed the corresponding technologies:

  1. Java

    $ java -version

  2. Maven

    $ mvn -v

Installation

View the source of this content.

Heads up!
	If your build fail, try the same command again.
	Sometimes the network brings up errors when
	requesting online dependencies

  1. Download the sources from the GitHub repository

    git clone -b "master" https://github.com/DanBeltF/reactivexd.git

  2. Build the language

    cd co.edu.escuelaing.reactivexd.parent-master
    mvn clean install -DskipTests=true

Build the sample projects

Hardcoded messages

This example executes the test directory that prints hardcoded messages.

  1. Go back to the root directory

    cd ..
    cd test
    cd /Hello-World/

  2. Run the application's test

    mvn clean test

Semi-smooth messages

In this example there are events which are exchanged, they take place in a non-controled context where a sequence of messages is followed. To simulate this use case, there will be two programs that interact with the JBossCache, used to save and read data.

  1. Go back to the root directory

    cd ..
    cd /Smooth-World/

  2. Compile and generate the sources

    mvn clean compile

  3. Run the generated application

    mvn exec:java -Dexec.mainClass="local.StartExecute"

  4. Open another command line and run

    mvn exec:java -Dexec.mainClass="local.RunExecute"

Group Chat

Finally a group chat example is presented by sending messages from one node (terminal console/computer/server) to another.

  1. Go back to the root directory

    cd ..
    cd /GroupChat/

  2. Compile and generate the sources

    mvn clean compile

  3. Run the application on two or more terminals

    mvn exec:java

  4. Add a name in order to identify the terminal and exchange messages

  5. Type end to finish the session
